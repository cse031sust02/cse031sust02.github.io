Understanding the Nginx Server Block

Sep 19, 2017 #nginx

We can think of Server Blocks as specifications for individual web sites. Server blocks are the NGINX equivalent of Apache’s virtual hosts. We need to understand the directives and settings that make up the server block.

Here is a sample virtual host :

server {
  listen       80;
  server_name  _;

  location / {
    root   /usr/share/nginx/html;
    index  index.html index.htm;
  }

  error_page  404              /404.html;
  location = /404.html {
    root   /usr/share/nginx/html;
  }
}

We can see different types of directives such as listen, root, server_name, location etc. Let’s explore those directives.

listen

It specifies the IP address / port combination that this server block is designed to respond to.

We can specify only the port or only the address or Both.

If a port is omitted, the standard port(80) is used. And if an address is omitted, the server listens on all addresses. We can also use more than one listen directive, if needed. Here are some examles.

port is omitted :

listen     127.0.0.1; 
# will only respond to 127.0.0.1:80

address is omitted :

listen     50; 
# will respond to port 50 of all addresses. i.e: 127.0.0.1:50, localhost:50 etc..

IP address / port combination :

listen     192.168.25.19:60; 
# will only respond to 192.168.25.19:60

server_name

Nginx also checks the Host header of each request. When client makes a requests, Nginx checks if there is any server block that has a server_name directive defined with that “Host”. If there is a match, then that server block will responds to that request.

It will be easy to understand with an example.

Let’s create two new host in the /etc/hosts file :

127.0.0.1   localhost
127.0.0.1   www.site-one.com
127.0.0.1   www.site-two.com

So, now both site-one and site-two points to same IP address. But we want to put one website in site-one.com and another website in site-two.com. Let’s create two different server blocks for those two hosts :

# First Website
server {
  listen       70;
  server_name  www.site-one.com;
  
  root   /usr/share/nginx/html/site-one/;
}

# Second Website
server {
  listen       70;
  server_name  www.site-two.com;

  root   /usr/share/nginx/html/site-two/;
}

It is clear that we can browse First Website with the address www.site-one.com:70, and the Second Website with www.site-one.com:70. Now, there is a Quiz. What website will open when we go to 127.0.0.1:70?

root

This directive specifies the directory where the website’s contents are located. So, in our last example, If the URL is http://www.site-two.com/mypage.html, Nginx will load mypage.html file from /usr/share/nginx/site-two directory.
